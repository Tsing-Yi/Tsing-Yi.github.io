What Is POD

Plian Old Data:通常用于说明一个类型的属性,尤其是用户自定义类型.

  • Plain表示了POD是一个普通的类型,C++常见类型都有类似属性,不像存在着虚函数和虚继承的类型那么特别.
  • Old体现了与C的兼容性,比如可以用memcpy进行复制,用memset进行初始化等.

Why POD

  • 字节赋值:代码中可以安全的使用memcpy和memset对POD类型进行初始化和拷贝等操作;
  • 对C内存布局兼容:POD类型的数据在C和C++间的操作总是安全的;
  • 保证静态初始化的安全有效:静态初始化在很多时候能够提高程序性能,而POD类型对象的初始化往往更加简单.

How To Define POD

C++11将POD划分为两个概念的合集trivial和standad layout.

Trivial

  • 拥有平凡的trivial constructor和平凡的trivial destructor
    通常情况下,不定义构造函数,编译器就会默认生成一个trivial constructor.而一旦定义了构造函数,即使不包含参数,函数体内也没有任何代码,该构造函数也不是trivial;
    析构函数也是如此;
    可以使用= default将构造函数和析构函数定义为trivial.

  • 拥有平凡的trivial copy constructor和平凡的trivial move constructor
    trivial copy constructor基本等同于使用memcpy进行构造;
    若不声明编译器会自动生成,或者使用= default;
    trivial move constructor与trivial copy constructor相似,只不过使用的移动语义.

  • 拥有平凡的trivial assignment operator和平凡的trivial move operator
    与上述类似.

  • 不能包含虚函数以及虚基类

Standad Layout

  • 所有非静态成员拥有相同的访问权限(private,protect,public)

  • 在类或结构体赋值时,满足一下两种情况之一:
    派生类中有非静态成员,且只有一个仅包含静态成员的基类;
    基类有非静态成员,派生类没有非静态成员.

  • 类中的第一个非静态成员的类型与基类不同.

该规则是基于C++中不允许优化不包含成员的基类而产生的.

  • 没有虚函数和虚基类

  • 所有非静态成员均符合标准布局类型,其基类也符合标准布局.

What Is RAII

Resource Acquistion Is Initialzation:资源获取即初始化,C++语言中的一个惯用法,用于管理资源,避免内存泄漏.

How To Implement RAII

构造一个类,在构造函数中申请资源,控制对资源的访问使之在对象的生命周期内始终有效,在析构函数中释放资源.

  • 不需要显式地释放资源;
  • 对象所需的资源在生命周期内始终有效;
  • 避免内存泄漏.

What Is Resource

如内存,文件句柄,网络套接字(network sockets),互斥锁(mutex lock)等等.

What Is Big/Little-Endian

Big-Endian

数据的高字节保存在内存的低地址中,而数据的低字节保存在内存的高地址中.

little-Endian

数据的低字节保存在内存的高地址中,而数据的高字节保存在内存的低地址中.

How To memory

小端低低,小端与人的逻辑顺序一致.

Why Big/Little-Endian

在操作系统中,x86和一般的OS（如windows,FreeBSD,Linux）使用的是小端模式.但比如Mac OS是大端模式.

在计算机系统中,我们是以字节为单位的,每个地址单元都对应着一个字节,一个字节为8bit.但是在C语言中除了8bit的char之外,还有16bit的short型,32bit的long型（要看具体的编译器）.另外,对于位数大于8位的处理器,例如16位或者32位的处理器,由于寄存器宽度大于一个字节,那么必然存在着一个如果将多个字节安排的问题.因此就导致了大端存储模式和小端存储模式.

  • 不同端模式的处理器进行数据传递时必须要考虑端模式的不同.

  • 在网络上传输数据时,由于数据传输的两端对应不同的硬件平台,采用的存储字节顺序可能不一致.所以在TCP/IP协议规定了在网络上必须采用网络字节顺序,也就是大端模式.对于char型数据只占一个字节,无所谓大端和小端.而对于非char类型数据,必须在数据发送到网络上之前将其转换成大端模式.接收网络数据时按符合接受主机的环境接收.
